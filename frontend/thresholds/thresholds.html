Configure threshold values for sovereignty characteristics and their lifecycle criteria

About Thresholds:

Thresholds define the minimum acceptable values for each SLC criterion within a Sovereignty Characteristic (SC) when marked as SHALL.

Each SC can have different thresholds for the same SLC, allowing fine-grained control over sovereignty requirements.

Threshold History: When you change a threshold and download the configuration, the new value is prepended to the history. The most recent value (first in the array) is always used for evaluation.

Save Changes: Changes are local until you download the configuration file. Replace the existing thresholds file with the downloaded one to persist your changes.
